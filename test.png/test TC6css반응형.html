  • RWD반응형

뷰포트란 무엇입니까?


뷰포트는 웹 페이지에서 사용자가 볼수 있는 영역입니다.
뷰포트는 장치에 따라 다르며 컴퓨터 화면보다 휴대 전화에서 더 작습니다.
태블릿과 휴대 전솨 이전에는 웹 페이지가 컴퓨터 화면 전용으로 설계되었으며 웹 페이지는 정적 디자인과 고정 크기를 갖는 것이 일반적이 었습니다.

그런다음 태블릿과 휴대 전화를 사용하여 인터넷 서핑을 시작 했을 때 고정 크기 웹 페이지가 너무커서 뷰포트에 맞지 않았습니다.
이 문제를 해결하기 위해 해당 장치의 브라우저는 전체 웹 페이지를 화면에 맞게 축소 했습니다.

뷰포트 설정

HTML5는 웹 디자이너가 <meta>태그를 통해 뷰포트를 제어 할수 있는 방법을 도입했습니다.



<meta>모든 웹 페이지에 다음 뷰포트 요소를 포함해야 합니다.
<meta name="viewport:" content="widht=device-width,initial-scale=1.0"> 뷰포트 요소는 페이지의 크기와 스케일링을 제어 하는 방법에 브아우저 지침을 제공 합니다. 이 width=device-widht 부분은 장치의 화면 너비에 따라 페이지 너비를 설정합니다. initial-scale=1.0부분은 페이지가 처음 브라우저에 의해 로드 된 초기줌 레벨을 설정 합니다. 다음은 뷰포트 메타 태그가 없는 웹 페이지 와 뷰포트 메타 택가 있는 동일한 웹 페이지의 예 영역입니다. 팁;휴대 전화나 태블릿으로 페이지를 탐색하는 경우 아래 두 링크를 클릭하여 차이점을 확인 할수 있습니다.

뷰포트에 내용 크기

사용자는 데스크탑 과 모바일 장치 모두에서 웹 사이트를 세로로 스크롤 하지만 가로로는 사용하지 않았습니다.
따라서 사용자기 전체 웹 페이지를 보기 위해 가로로 스크롤 하거나 축소하면 사용자 환경이 저하됩니다.

따라야할 몇가지 추가 규칙

  • 큰 고정 너비 요소를 사용하지 마십시요.

    • 이미지가 뷰포트 보다 넓은 너비로 표시되면 뷰포트가 가로로 스크롤 될수 있습니다.
    뷰 포트 너비에 맞게 조정 해야 합니다.

  • 내용을 잘 렌더링 하기 위해 특정 뷰포트 너지에 의존하지 않도록 하십시요.

    • CSS 픽셀의 화면 크기와 너비는 장치마다 크게 다르므로 내용을 잘 렌더링 하기 위해 특정 뷰포트 너비에 의존해서는 안됩니다.

  • CSS 미디어 쿼리를 사용하여 작고 큰 화면에 다른 스타일 적용

    • 페이지 요소에 절대 CSS 너비를 크게 설정하면 요소가 더 작은 장치의 뷰포트에 비해 너무 넓습니다. width:100%와 같은 상대 너비 값을 사용하십시요. 위치 결정 값이 큰지 주의작은 장치에서는 요소가 뷰포트 밖으로 떨어질수 있습니다.

그리드 뷰란?
많은 웹 페이지는 격자보기를 기반으로 합니다.
페이지가 열로 나뉩니다.

격자보기를 사용하면 웹 페이지를 디자인 할때 매우 유용합니다.

12개의 열이 있으며
총 너비가 100%이며
브라우저 창의 크기를 조정하면 축소 및 확장됩니다.

반응형 그리드 뷰 작성

반응형 그리드 뷰 구축
모든 HTML요소의 box-sizing 속성으로 설정되어 잇는지 확인
border-box이렇게 하면 패딩 과 테두리 요소가 총 너비와 높이에 포함이 됩니다.

CSS 코드 추가

*{
box-siwing: border-box;
}
